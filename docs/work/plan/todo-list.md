# 무드미 Todo List

팀 전체 todo를 역할(아키타입)별로 관리하는 문서입니다.

- **항목 추가는 `/work-todo`로**: 내용(역할, 작업이름, 일정, 페이지)만 입력하면 번호가 자동 부여됩니다. **번호를 직접 매기지 마세요.**
- **작업이름·일정·페이지는 직접 수정해도 됩니다.** 번호와 이슈 열만 스킬이 관리합니다.
- **이슈 열은 `/work-issue <번호>` 가 자동 기록**합니다. 이슈 열이 채워진 항목은 이미 GitHub 이슈로 등록된 것입니다.
- 일정 형식: `26.07.06~26.07.12` / 페이지 값: `로그인 | 메인 | 보드생성 | 보드완성 | 보드편집 | 테스트 | 공통`

> **마지막 부여 번호: a1** — 번호는 `/work-todo`가 자동 부여. 직접 매기지 않는다. 삭제된 항목의 번호도 재사용 금지.
