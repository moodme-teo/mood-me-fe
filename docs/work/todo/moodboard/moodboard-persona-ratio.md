# 페르소나 산출 — 여정 → 두 축의 비율

> 📌 [moodboard-creation.md](./moodboard-creation.md) §2 "여정 → 페르소나 비율"의 본문. **이 파일이 산출 규칙의 정본이다.**
>
> 산출된 비율을 키워드로 바꾸는 조립 규칙은 `moodboard-creation.md` §3, 키워드 사전은 [moodboard-persona-list.md](./moodboard-persona-list.md), 프롬프트 템플릿은 [moodboard-image-prompt.md](./moodboard-image-prompt.md) 에 있다.

## 산출 방식

이미지 생성에 앞서, 테스트 여정만으로 페르소나를 정한다. **AI 없이 사용자 선택만으로 계산** — AI는 이 결과를 받아 해석·프롬프트를 만들 뿐, 페르소나 판정 주체가 아니다.

- 구현: **`src/lib/mood-test/persona.ts`** — `computePersonaResult(journey): PersonaResult` 순수 함수.
- 입력: `Journey`(id 기반, `lib/mood-test/journey.ts`). 출력: 코어/테마 두 축의 페르소나 순위(`PersonaRank[]`).

### 핵심 — 하나의 비율이 아니라 2개 축

두 축은 성격이 달라 **같은 저울에 세우지 않는다** — 각 축에서 따로 순위를 매긴다. 1등끼리 합쳐 유형명을 만들지는 않는다(§개요 주).

| 축        | 재는 것                         | 주 측정원                        | 개수 |
| --------- | ------------------------------- | -------------------------------- | ---- |
| 미학 코어 | 어떻게 **보이고** 싶나 (비주얼) | 카드 personaWeights              | 14종 |
| 인생 테마 | 어떻게 **살고** 싶나 (태도)     | 전환(그림자→열망) + 카드 theme형 | 6종  |

### 가중치 (persona.ts 상수 — 튜닝 지점)

**카드 = 운명(fate) 배수** — 늦게까지 살아남을수록 강한 신호:

| 카드의 운명                           | 배수                                             |
| ------------------------------------- | ------------------------------------------------ |
| final에 남은 확신 (kept_conviction)   | **×3.0**                                         |
| 열망에 밀린 확신 (dropped_conviction) | **×1.5**                                         |
| 늦은 탈락 dropped_r2 (미련)           | **×0.5**                                         |
| 이른 탈락 dropped_r1                  | **×0.0** (확신만 반영. 넓은 그물까지 보려면 0.2) |

**전환(열망) = 테마 가중**:

| 열망의 운명                          | 가중                                            |
| ------------------------------------ | ----------------------------------------------- |
| final까지 남은 열망 (kept_desire)    | **+1.0**                                        |
| final에서 밀린 열망 (dropped_desire) | **+0.5**                                        |
| ★뻔한 반대말(t*-1)                   | **+0.0** (테마 무관 — "정면돌파형" 톤 신호로만) |

**확정된 노브** (2026-07-09):

- `toggles`(망설임)는 **점수에 넣지 않음** — 서사 참고용으로만.
- 이른 탈락(dropped_r1)은 **0** — 순수하게 확신만 반영.
- 그림자(C)는 **자체 페르소나 가중 없음** — "보편 결핍"이라 페르소나 무관(mood-test-questions.md). 그림자는 자신이 여는 **전환을 통해서만** 정체성에 기여한다.

### 입력 여정(journey) 예시

`computePersonaResult`의 유일한 런타임 입력은 완료된 `Journey`(id 기반) 하나다. 아래는 아래 워크드 예시(코스탈 × 코지 홈바디)를 만든 **실제 유효 여정** — `journeySchema` 길이·분할·1:1 제약을 모두 통과한다.

```jsonc
{
  // A. 담은 12장 = dropped_r1 + dropped_r2 + survivors (집합 일치)
  "selected": [
    "c17",
    "c20",
    "c19",
    "c35",
    "c36",
    "c02",
    "c04",
    "c13",
    "c05",
    "c06",
    "c11",
    "c12",
  ],
  "dropped_r1": ["c02", "c04", "c13", "c05"], // B1 이른 탈락 4 (×0)
  "dropped_r2": ["c06", "c11", "c12"], // B2 늦은 탈락 3 (×0.5)
  "survivors": ["c17", "c20", "c19", "c35", "c36"], // 확신 5
  "shadows": ["s5", "s1", "s8"], // C 그림자 3 (점수 미사용)
  "transitions": [
    // D 전환 — picked만 점수에 씀
    { "shadow": "s5", "picked": "t5-1" }, // 여유 ★
    { "shadow": "s1", "picked": "t1-1" }, // 휴식 ★
    { "shadow": "s8", "picked": "t8-1" }, // 고요 ★
  ],
  "final": ["c20", "c35", "c36", "t5-1", "t1-1"], // E 최종 5 = 남긴 확신 3 + 남긴 열망 2 (id 혼합)
  "dropped_final": ["c17", "c19", "t8-1"], // E 밀림 3 = 밀린 확신 2 + 밀린 열망 1
  "toggles": {}, // 망설임 — 점수 미사용
  "toggle_count": 0,
}
```

카드 라벨: c17 숲속 빛내림·c20 모래 맨발·c19 파도 거품·c35 고양이 낮잠·c36 홈카페 라떼(확신) / c02 정돈된 화장대·c04 요트 린넨·c13 크림톤 노트·c05 발레슈즈(이른 탈락) / c06 딸기케이크·c11 촛불 고서·c12 도서관 사다리(늦은 탈락).

**필드 → 산출 반영** (`c##`/`t#-#` 접두가 아니라 `survivors`·`transitions` 소속으로 카드/열망을 가른다):

| 여정 필드             | 값             | 산출 반영            |
| --------------------- | -------------- | -------------------- |
| `survivors` ∩ `final` | c20·c35·c36    | kept 확신 **×3.0**   |
| `survivors` − `final` | c17·c19        | 밀린 확신 **×1.5**   |
| `dropped_r2`          | c06·c11·c12    | 늦은 탈락 **×0.5**   |
| `dropped_r1`          | (4장)          | **×0** 무기여        |
| `transitions.picked`  | t5-1·t1-1·t8-1 | 전부 ★ → 테마 **+0** |

### 워크드 예시 (검증 완료)

입력: 위 여정 — 코스탈·코지 계열 확신(모래·파도·고양이·홈카페·숲) + 전환 3개 모두 ★뻔한 반대말(여유·휴식·고요). **전환이 전부 ★라 테마 기여 0 → 테마는 카드만으로 잡힌 엣지 케이스.**

**미학 코어** (총 8.2점)

| 페르소나          | 점수 | 비율 |
| ----------------- | ---- | ---- |
| 코스탈 🏆         | 3.3  | 40%  |
| 페어리코어        | 1.2  | 15%  |
| 라이트 아카데미아 | 1.1  | 13%  |
| 클린 걸           | 0.9  | 11%  |
| 다크 아카데미아   | 0.7  | 9%   |
| 코티지코어        | 0.6  | 7%   |
| 코케트            | 0.3  | 4%   |
| 네오 로맨틱       | 0.1  | 1%   |

**인생 테마** (총 5.3점)

| 페르소나       | 점수 | 비율 |
| -------------- | ---- | ---- |
| 코지 홈바디 🏆 | 4.7  | 89%  |
| 스피리추얼     | 0.3  | 6%   |
| 트래블러       | 0.3  | 6%   |

**⇒ 코어 1등 코스탈 40% · 테마 1등 코지 홈바디 89%** (persona.ts 실행 결과와 일치 확인)

### 열린 문제 — ★만 고른 유저의 테마

유저가 전환 3개를 전부 ★로 고르면 전환의 테마 기여가 0이 된다. 위 예시는 카드 theme형(코지 홈바디)이 받쳐줘 문제없었지만, 카드에 theme형이 하나도 없으면 테마 자리가 빌 수 있다. **권장: ★는 "정면돌파형" 톤 태그로만 쓰고 테마는 카드에 위임** (극단 케이스는 실제로 거의 안 나옴). 확정 필요.
