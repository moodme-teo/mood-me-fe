# 보드 레이아웃 스타일 사전

> 📌 [moodboard-image-prompt.md](./moodboard-image-prompt.md) 템플릿의 `{LAYOUT_STYLE_BLOCK}` 자리에 들어가는 값. **이 파일이 레이아웃의 정본이다.**
>
> 페르소나 비율이 "무엇을 그릴지"를 정한다면, 레이아웃 스타일은 **"어떻게 배치할지"** 를 정한다. 두 축은 독립이다 — 같은 페르소나 결과라도 레이아웃이 달라지면 다른 보드가 나온다.

## 레이아웃이 템플릿보다 우선한다

**레이아웃 스타일은 프롬프트의 최상위 권한이다.** 템플릿의 다른 지시와 어긋나면 레이아웃이 이긴다. 템플릿에는 이 문장이 `{LAYOUT_STYLE_BLOCK}` 바로 아래에 박혀 있다.

```
위 레이아웃 스타일은 이 프롬프트의 다른 모든 지시보다 우선한다.
아래 지시가 레이아웃 스타일과 어긋나면, 아래 지시를 버리고 레이아웃 스타일을 따른다.
레이아웃 스타일이 "넣지 마라"고 한 요소는 아래에서 요구하더라도 넣지 않는다.
```

그래서 **조각 수 · 텍스트 조각 수 · 타이틀 카드 유무 · 여백 허용은 템플릿에서 뺐고, 각 레이아웃이 직접 선언한다.** 템플릿은 레이아웃이 값을 안 주는 경우의 기본값(`텍스트 3~6개` · `최대 50자`)만 갖는다.

레이아웃과 무관해 템플릿에 남은 규칙: 페르소나 이름 금지 · 목록/불릿 금지 · 왜곡된 얼굴 금지 · 팔레트 결정권 · 비율 반영 밴드.

## 왜 필요한가

템플릿이 배치까지 한 가지로 못박고 있어서, **같은 여정이면 매번 거의 같은 그림이 나왔다.** v5(코스탈)와 Y2K 회차는 페르소나가 완전히 달랐는데도 "중앙 타이틀 + 빽빽한 겹침 스크랩북"이라는 골격이 똑같았다. 팔레트와 소재만 갈리고 구조는 한 벌뿐이었다.

## 실측 (2026-07-09) — 올드 머니 여정 8종 비교

같은 페르소나 비율(올드 머니 52% · 커리어 보스 46%)로 레이아웃만 바꿔 8장을 뽑았다.

- **레이아웃은 확실히 구조를 가른다.** A/B 대조에서 레이아웃 절을 넣은 쪽만 배경이 드러나고 별 스티커가 나왔다. 더 나아가 **A가 빠뜨린 `tennis club`을 B가 그렸다** — 여백이 생기자 꼬리 키워드가 들어갈 자리가 났다. 레이아웃은 키워드 보존에도 영향을 준다.
- **팔레트는 레이아웃이 못 이긴다.** 8종 전부 올드 머니의 뉴트럴 톤을 유지했다. `taped_paper_pileup`의 차콜 배경 위 조각들조차 베이지·골드였다. 팔레트 결정권(코어 1위)은 레이아웃보다 위다.
- **재현도**: `taped_paper_pileup`·`sparse_scrapbook` 최상 → `center_title_shaped_cutout`·`object_bridged_fullbleed` 상 → `large_image_patchwork`·`image_first_minimal` 중 → `text_capture_object_mix` 하.

### ⚠️ `[예외]` 방식이 실패한 기록 — 우선순위 선언으로 교체한 이유

처음엔 템플릿 규약을 그대로 두고 레이아웃 블록에 `[예외] …`를 적어 덮어쓰려 했다. **절반만 먹혔다.**

| 예외의 종류                     | 결과    | 사례                                                                          |
| ------------------------------- | ------- | ----------------------------------------------------------------------------- |
| 개수를 바꿔라 (3~6 → 10개, 2개) | ✅ 먹힘 | `taped_paper_pileup` 라벨 10개 · `center_title…` 메모 2개                     |
| 금지를 풀어라 (액자·여백·50자)  | ✅ 먹힘 | `sparse_scrapbook` 금색 액자 · `object_bridged…` 51자 문장                    |
| **요소를 없애라 (타이틀 카드)** | ❌ 실패 | `image_first_minimal`·`text_capture…`·`object_bridged…` 셋 다 타이틀이 그려짐 |

**"넣어라"는 지시는 예외로 못 이긴다.** 템플릿의 `중앙에 타이틀 카드를 하나 배치한다`가 세 번 다 이겼다. `image_first_minimal`은 다른 글자를 전부 지우고도 타이틀만은 남겼고, `object_bridged_fullbleed`는 풍선 숫자를 타이틀 카드 _위에_ 얹는 식으로 절충했다.

→ 그래서 예외 표기를 버리고, **템플릿에서 해당 규칙 자체를 빼 레이아웃에 위임**하고 우선순위를 명문화했다. `text_capture_object_mix`가 SNS 캡처 1장만 그리고 손글씨 메모로 회귀한 것도, 템플릿의 `텍스트는 손글씨처럼 보여야 한다`가 살아 있었기 때문이다.

## 선택 방식 — 여정 해시 (확정 2026-07-09)

**여정을 해시해 7종 중 하나를 고르고, 고른 `layout_id`를 보드 row에 저장한다.** 페르소나와 연동하지 않는다.

```
layout_id = LAYOUTS[ hash(canonical(journey)) % LAYOUTS.length ]
```

- `canonical(journey)` = `final` · `shadows` · `transitions.picked`를 각각 **정렬**해 이어붙인 문자열. 카드를 담은 **순서는 넣지 않는다** — 같은 선택을 다른 순서로 담았다고 구조가 바뀌면 안 된다.
- 해시는 결정론적이면 무엇이든 된다 (FNV-1a 등). 암호학적 성질은 필요 없다.

### 왜 페르소나 연동이 아닌가

- **동점을 처리할 수 없다.** 무작위 여정 5,000개 실측에서 **코어 1·2위 동점이 10.4%**다. 팔레트는 균등 융합으로 풀지만 레이아웃은 융합이 안 된다 — 스크랩북과 미니멀을 반씩 섞을 수 없다. 결국 배열 순서로 가르게 되는데, 그건 팔레트에서 이미 폐기한 방식이다.
- **같음이 증폭된다.** 코어 1위가 팔레트 주인이자 레이아웃 결정자가 되면 같은 코어끼리 **색도 구조도 같은** 보드를 받는다. 지금은 두 축이 독립이라 색이 같아도 구조는 갈린다.
- **일부 스타일이 사장된다.** 코어 1위 분포는 코티지코어 10.4% ~ 올드 머니 3.2%로 3.3배 벌어진다(실측, 무작위 여정 기준이라 실사용자는 더 심할 것). 희귀 코어에 걸린 레이아웃은 서른 명에 한 명만 본다.

### 왜 저장하는가

`% 7`은 **스타일 개수가 바뀌면 전원이 재배치된다.** 8종으로 늘리는 순간 과거 여정의 레이아웃이 전부 달라져, 결정론적이라는 장점이 버전 간에는 깨진다. `layout_id`를 저장해두면 해시는 **최초 생성 때 한 번만** 쓰이고, 재생성은 저장값을 읽는다. 어떤 레이아웃의 반응이 좋은지 집계할 수도 있다.
