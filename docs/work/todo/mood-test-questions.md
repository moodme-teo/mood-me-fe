## 설계 사상 — 질문이 아니라 메커니즘

단발 객관식은 "무엇을 좋아하나"의 태그만 남기지만, 이 테스트는 **행동 데이터**를 만듭니다:

1. **희소성 강제 덜어내기** — 담은 카드를 두 번 내려놓게 한다. 무엇을 언제 포기했는지(탈락 라운드)가 신호.
2. **그림자 직면** — 원하는 것만 묻지 않고 "요즘 나를 무겁게 하는 것"을 따로 묻는다. 결핍은 열망보다 정직하다.
3. **반대말을 스스로 고르게 하기** — 그림자마다 정답 없는 4지선다(뻔한 반대말 1 + 결이 다른 해석 3). 어떤 해석을 고르는지가 그 사람 고유의 프레임을 드러낸다.
4. **확신 vs 열망의 최종 대결** — 살아남은 카드(확신)와 전환 선택(열망)을 한 판에 올려 최종 5장을 고르게 한다. 열망이 확신을 밀어냈는지가 마지막 신호.
5. **여정 전체를 해석 프레임과 함께 Claude에 전달** — 최종 5장만이 아니라 전 과정 로그를 준다.

**무게 조절**: 메커니즘은 유지하되 카피는 놀이처럼 가볍게(PRODUCT.md). "내려놓아 주세요" 같은 부드러운 명령형은 살리고, 그림자는 일상어로 낮춘다.

## 테스트 플로우 (`/test/:sessionId`, 목표 2~3분) — ✅ 구성 확정

| 단계 | 화면 | 동작 | 수집 데이터 |
| --- | --- | --- | --- |
| A. 담기 | 카드 그리드 (36장, 3열) | **"끌리는 카드 12장을 담아주세요"** — 생각은 넣어두고, 손이 먼저 가는 걸로요 | `selected[12]` |
| B1. 덜어내기 | 담은 12장만 표시 | **"이 중 4장을 내려놓아 주세요"** — 아깝죠? 괜찮아요, 남는 게 더 선명해져요 → 8장 | `dropped_r1[4]` |
| B2. 더 덜어내기 | 8장 | **"3장만 더 내려놓을게요"** — 지금 남은 5장이 당신의 확신이에요 → 5장 = **확신의 가치** | `dropped_r2[3]` |
| C. 그림자 | 텍스트 칩 8개 | **"요즘 나를 무겁게 하는 것 3개를 골라주세요"** — 솔직할수록 보드가 정확해져요 | `shadows[3]` |
| D. 전환 | 그림자마다 4지선다 ×3 | **"'{그림자}'를 넘어서기 위해, 지금 당신에게 필요한 힘은?"** (그림자 동적 삽입) = **열망의 가치** | `transitions[3]` |
| E. 최종 선택 | 확신 5장 + 열망 3장 = 8장 그리드 | **"마지막이에요 — 당신의 무드보드에 남길 5장을 골라주세요"** — 지켜온 것과 바라는 것, 무엇이 남을까요 | `final[5]`, `dropped_final[3]` |

- **프리뷰 연출**: 덜어내기는 상실이 아니라 확정의 경험으로 — 상단 프리뷰 보드에서 카드가 빠지는 게 아니라 **남은 카드가 자리를 잡아가는** 연출 (PRD 5.3 F-07의 "채워지는" 경험과 결합).

## 로깅 스키마 — 전 과정 로깅이 곧 제품

리포트의 "○○는 마지막까지 붙들다 놓았다" 같은 문장은 이 JSON 없이는 원리적으로 불가능합니다. 프론트 이벤트만 쌓으면 되므로 구현 비용은 낮습니다.

```jsonc
{
  "selected": ["c07", "c12", ...],          // A에서 담은 12장 (담은 순서 유지)
  "dropped_r1": ["c12", ...],               // B1 탈락 4장
  "dropped_r2": ["c07", ...],               // B2 탈락 3장 (늦게 탈락 = 미련 큼)
  "survivors": ["c03", ...],                // 확신의 가치 5장
  "shadows": ["s2", "s5", "s7"],            // C 선택 3개
  "transitions": [                           // D — 그림자별 선택
    { "shadow": "s2", "picked": "t2-3" }, ...
  ],
  "final": ["c03", "t2-3", ...],            // E 최종 5 (확신/열망 혼합)
  "dropped_final": [...],                    // E에서 밀려난 3
  "toggles": { "c07": 2, "c12": 1 },        // 카드별 담았다 뺐다 횟수 (망설임 신호)
  "toggle_count": 3                          // 합계 (파생값)
}
```

- `toggles`는 **카드별 맵**으로 기록 — "'손편지'를 두 번 뺐다가 결국 담았다" 같은 리포트 문장의 재료. 구현은 이벤트 카운터 객체 하나. 활용 방식(리포트 반영 강도)은 확정 전 — 우선 데이터는 쌓는다. `dwell_ms`(단계별 체류시간)는 nice-to-have — 여유 있을 때.
- 이 JSON이 `MoodTestSession`의 답변 스키마가 됩니다 (빌더 확인 필요).

카드 풀·결과 유형의 기준 좌표계. 리서치(비전보드 신 = 미학 코어 × 인생 테마) 기반 — 테스트 중에는 장면 카드만 보여주고, 페르소나 이름은 결과 유형명에서만 사용한다.

- **결과 유형 = 미학 코어 1 + 인생 테마 1 조합** (예: "다크 아카데미아 × 커리어 보스", 84조합) — 생존 카드 + 최종 5장의 페르소나 가중 합산으로 산출.
- **배분**: 카드 36장 = 미학 코어 14종 × 2장(28) + 인생 테마 6종 × 1~2장(8). 카드마다 페르소나 2~3개가 겹치게 — 카드 한 장이 아니라 조합이 판별한다.
- **판별 쌍** (가르는 카드 필수): 코케트↔네오 로맨틱(파스텔 리본 vs 레이스 시어) · 페어리↔코티지(빛·몽환 vs 부엌·노동) · 클린 걸↔웰니스(정돈 vs 땀·기록) · Y2K↔맥시멀 글램(플라스틱 원색 vs 텍스처 드라마)
- 카드 이미지 스타일: **감성 디테일 컷** — 스타일 이름이 아니라 한 장면·한 사물 ("싱크대에 꽂은 꽃" 급). 셀럽은 초상권 문제로 익명 실루엣/뒷모습만.

그림자마다 **★뻔한 반대말 1개(중립 기준점) + 결이 다른 해석 3개** (구조 확정). 여기에 **인생 테마 축을 심는다** — 카드 36장이 미학 코어 축을 재고, 전환 선택지가 인생 테마 축을 잰다. 선택지 라벨은 최종 대결(E)에 열망 카드로 다시 등장하므로 짧은 가치 언어로:

| 그림자 | 선택지 (★ = 뻔한 반대말 · 괄호 = 테마 가중) |
| --- | --- |
| s1 번아웃 | ★휴식 · 몰입 (커리어 보스) · 활력 (웰니스 러너) · 훌쩍 떠나기 (트래블러) |
| s2 비교하는 마음 | ★자존감 · 나만의 속도 (웰니스 러너) · 나의 세계 (홈바디·취향) · 행운의 감각 (러키걸) |
| s3 미루는 버릇 | ★실행력 · 가벼운 시작 (웰니스·루틴) · 설레는 목표 (러키걸) · 짜릿한 마감 (커리어 보스) |
| s4 눈치보기 | ★당당함 · 솔직함 (표현) · 나의 편 (코지 홈바디) · 쿨한 거리 (미니멀) |

- id 체계: `t{그림자번호}-{1~4}`, `t{n}-1`이 항상 ★.
