# PRD: 추구미 무드보드 서비스

## 1. 개요

### 1.1 배경

사용자가 자신이 "추구하는 분위기(추구미)"를 이미지·키워드 선택 기반의 간단한 테스트로 표현하면, AI가 이를 시각적 무드보드로 자동 생성해주고 사용자가 직접 편집·완성하여 SNS에 공유할 수 있는 서비스.

### 1.2 용어 정의

| 용어      | 정의                                     |
| :------ | :------------------------------------- |
| 추구미     | 사용자가 지향하는 스타일/분위기 (미적 취향)              |
| 추구미 테스트 | 이미지/키워드 선택을 통해 사용자의 취향을 수집하는 질문 플로우    |
| 무드보드    | 선택 결과를 바탕으로 AI가 생성한, 이미지·색·키워드로 구성된 보드 |
| 비회원     | 카카오 로그인을 하지 않고 서비스를 이용하는 사용자           |

## 2. 목표

* 사용자가 로그인 장벽 없이 즉시 무드보드 생성 경험을 체험할 수 있게 한다.
* AI 생성 대기 시간을 지루하지 않게 만들어 이탈을 최소화한다.
* 결과물(무드보드 + 성향 그래프)의 SNS 공유를 유도해 자연 유입을 만든다.

### 성공 지표 (제안, 추후 확정 필요)

* 추구미 테스트 완료율 (테스트 시작 → 생성 요청)
* 무드보드 저장/내보내기 전환율
* SNS 공유율
* 비회원 → 회원 전환율 (저장/공유 시점 로그인 유도 기준)

## 3. 대상 사용자 및 접근 권한

| 기능                | 비회원               | 회원(카카오/구글 SSO) |
| :---------------- | :---------------- | :------------- |
| 메인 페이지 열람         | ✅                 | ✅              |
| 추구미 테스트 진행        | ✅                 | ✅              |
| 무드보드 생성           | ✅                 | ✅              |
| 무드보드 편집           | ✅                 | ✅              |
| 결과 열람 (당일 세션 내)   | ✅                 | ✅              |
| 무드보드 영구 저장/재방문 조회 | ❌ (세션 종료 시 소실 가능) | ✅              |
| SNS 공유 / 이미지 내보내기 | ✅                 | ✅              |

> 비회원도 전체 플로우(테스트→생성→편집→결과·공유·내보내기)를 끝까지 이용 가능하게 하되, 결과물을 나중에 다시 보려면 로그인이 필요하다는 점을 결과 페이지에서 안내한다.

## 4. 전체 유저 플로우

```
[앱 실행 · 게스트 우선 진입]   ← 로그인 강제 없음
        │                      (프로필 버튼 → 로그인 화면 / 로그인 시 로그아웃 메뉴)
        ├─ 저장된 보드 있음 ──▶ [홈(History) · 저장 보드 목록] ──┐
        │                                                          │ (무드보드 만들기)
        └─ 저장된 보드 없음 ──▶ [메인 페이지] ─────────────────────┤
                                                                   ▼
                                                            [추구미 테스트]
                                                                   │ (3막 선택 진행)
                                                                   ▼
                                                          [무드보드 생성중]
                                                                   │ (AI 생성 완료)
                                                                   ▼
                                            (이탈 확인 → 홈) ◀── [무드보드 편집]
                                                                   │ (완성하고 공유하기)
                                                                   ▼
                                                          [최종 결과물 페이지]
                                                          ├─ SNS 공유
                                                          ├─ 이미지 내보내기
                                                          ├─ 편집으로 돌아가기
                                                          ├─ 처음부터 다시 만들기
                                                          └─ 홈으로

[홈(History)] ──(저장 보드 카드 탭)──▶ 해당 보드 결과 화면 열람
```

## 5. 화면별 상세 명세

### 5.1 진입 및 로그인 (게스트 우선)

> **정책:** 첫 진입에 로그인을 강제하지 않는다. 앱을 열면 곧바로 메인/홈으로 진입하며(게스트 우선), 로그인은 **사용자가 원할 때**만 노출한다.

* **진입 동작:** 앱 실행 → (온보딩 후) 저장된 무드보드가 있으면 홈(History), 없으면 메인으로 바로 이동. 이 시점의 사용자는 게스트 상태.
* **로그인 진입점:** 메인/홈 상단의 **프로필(아바타) 버튼**을 누르면 로그인 화면으로 이동한다. (첫 진입 강제 화면·모달 없음)
* **로그인 화면 구성:** 카카오 SSO, 구글 SSO, "로그인 없이 둘러보기(게스트)" 버튼
* **동작:**
  * 카카오/구글 로그인 성공 시 → 직전 화면(메인/홈)으로 복귀, 세션에 사용자 식별자 저장
  * 게스트 선택 시 → 게스트 세션을 유지한 채 메인/홈으로 복귀
* **로그인 상태 프로필:** 로그인 상태에서 프로필 버튼을 누르면 **계정 메뉴(로그아웃)** 가 노출된다.
* **예외:** 카카오/구글 인증 실패·취소 시 로그인 화면에 에러 토스트 노출, 재시도 가능

### 5.2 메인 페이지

* **경로:** `/`
* **구성:**
  * 서비스(무드보드) 설명 영역
  * 무드보드 예시 이미지 (캐러셀 또는 그리드, 다른 사용자 결과물 또는 큐레이션 샘플)
  * "무드보드 만들기" CTA 버튼
* **동작:** CTA 클릭 시 추구미 테스트 시작(담기 단계)으로 이동, 새 세션(테스트 세션) 생성
* **비회원 처리:** 별도 제약 없음

### 5.2-A 홈(History) — 저장된 무드보드 목록

> 저장된 무드보드가 1개 이상이면 진입 홈이 메인 대신 History가 된다. (저장 보드가 없으면 메인 페이지)

* **경로:** `/` (저장 보드 존재 시 기본 진입)
* **구성:**
  * 상단 "History" 타이틀 + 프로필(아바타) 버튼
  * "N개의 무드보드를 모았어요" 카운트
  * 저장된 무드보드 카드 목록 — 각 카드는 보드의 미니 콜라주 썸네일 + 성향 이름
  * "무드보드 만들기" CTA
* **동작:**
  * 카드 탭 → 해당 무드보드의 결과 화면(성향·그래프·공유)으로 이동해 열람
  * CTA → 추구미 테스트 시작(새 세션)
* **회원/게스트 차이:** 회원은 계정에 귀속된 보드가 표시(영구), 게스트는 세션 내 임시 보드만 표시(§5.7 참조). 로그아웃 시 초기 홈(메인)으로 이동.

### 5.3 추구미 테스트

> 상세 스펙(카드 36장·그림자 8개·전환 32개·카피·로깅 스키마·AI 해석 프레임): [docs/work/todo/mood-test-questions.md](../work/todo/mood-test-questions.md) — **7/7 확정, 이 문서가 구현 기준**

* **경로:** `/test/:sessionId`
* **구조:** 단발 객관식이 아닌 **3막 선택 메커니즘** (목표 2~3분) — 행동 데이터(무엇을 언제 버렸는지)가 곧 분석 재료
  1. **담기** — 카드 그리드 36장 중 끌리는 12장 담기
  2. **덜어내기 ×2** — 12→8→5장 강제 덜어내기. 남은 5장 = **확신의 가치**
  3. **그림자** — "요즘 나를 무겁게 하는 것" 칩 8개 중 3개
  4. **전환** — 그림자마다 4지선다(뻔한 반대말 1 + 결이 다른 해석 3). 선택 = **열망의 가치**
  5. **최종 대결** — 확신 5장 + 열망 3장 중 무드보드에 남길 5장 선택
* **상단 헤더:**
  * 뒤로가기 버튼 (이전 단계로, 첫 단계에서는 진입 화면(메인/홈)으로)
  * 진척도 표시 (단계 기준, 프로그레스바)
  * 프리뷰: 덜어내기에서 **남은 카드가 보드에 자리를 잡아가는** 연출 (상실이 아니라 확정의 경험). 점점 "완성되어 가는" 무드보드 미리보기를 실시간 반영.
    * **인터랙션:** "다음"을 누르면 해당 문항의 대표 항목(이미지 1점 또는 키워드 묶음)이 프리뷰 보드(점선 프레임 콜라주)로 **날아가 안착**하며 한 장씩 쌓인다. 문항당 대표 1점이 누적된다.
* **본문:**
  * 질문 텍스트 + 질문 설명(부가 설명/의도) + step 표시(step1…)
  * 선택지 UI 3가지 타입 중 질문별로 하나 적용
    1. **이미지만 선택** — 이미지 카드 그리드에서 1개 이상 선택
    2. **텍스트(키워드)만 선택** — 키워드 칩 목록에서 선택
    3. **이미지 선택 + 텍스트로 구체화** — 이미지 선택 후, 해당 이미지를 설명하는 키워드/짧은 텍스트 입력 또는 추가 선택
* **선택 규칙(다중 선택):**
  * 문항마다 **최소/최대 선택 수**를 둔다(단일 선택 문항 포함). 최소 미충족 시 "N개 이상 골라주세요 (선택/최소)" 안내, 최대 초과 시 "최대 N개까지 고를 수 있어요" 안내.
  * 조건 충족 전에는 "다음"이 비활성이며, 충족 시 "N개 선택됨" 상태를 표시한다.
* **여정 로깅:** `{selected, dropped_r1/r2, survivors, shadows, transitions, final, dropped_final, toggles}` — 전 과정이 세션에 저장되어 AI 분석 입력이 된다
* **동작:**
  * 선택 완료 시 "다음" 활성화 → 프리뷰 안착 애니메이션 후 다음 질문으로 이동, 진척도·프리뷰 갱신
  * **마지막 질문**에서는 "다음" 버튼이 **"무드보드 생성하기"**(스파클)로 전환된다.
  * 생성하기 → 무드보드 생성중 화면으로 이동, 지금까지의 선택 답변을 생성 요청으로 전송
* **상태 저장:** 뒤로가기로 이전 단계 이동 시 이전 선택값 유지 (재선택 가능)

### 5.4 무드보드 생성중

* **경로:** `/test/:sessionId/generating`
* **구성:**
  * 사용자가 선택한 이미지들이 가상의 무드보드 레이아웃 위에 순차적으로 배치되며 채워지는 인터랙션 애니메이션
  * 프로그레스바
  * 프로그레스바 상단 상태 메시지: "AI가 그리는 중..." 류의 위트있는 문구, 일정 간격(예: 2\~3초)마다 로테이션
* **동작:**
  * 백그라운드에서 실제 AI 생성 작업 진행 상태를 폴링/구독하고, 진행률에 맞춰 프로그레스바 갱신
  * 생성 완료 시 자동으로 무드보드 편집 화면으로 이동
* **예외:** 생성 실패 시 에러 화면 + "다시 시도" 버튼 (추구미 테스트 결과는 유지한 채 재생성 요청)

### 5.5 무드보드 편집

* **경로:** `/moodboard/:moodboardId/edit`
* **구성:**
  * AI가 완성한 무드보드 캔버스
  * 편집 도구: 이동(요소 선택·위치 이동/리사이즈), 스티커(스티커 팔레트에서 추가), 글자(텍스트 박스 추가/폰트·색상), 펜(자유 드로잉), **지우개(요소 탭하여 삭제)**
  * 저장(완성하고 공유하기) 버튼
* **동작:**
  * 각 도구는 캔버스 위 요소로 추가되며 개별 이동/삭제 가능. 요소 선택 시 삭제 버튼 노출, 지우개 모드에서는 탭으로 삭제.
  * 저장 시 현재 캔버스 상태(요소 좌표·속성 포함)를 서버에 저장 후 결과물 페이지로 이동
  * **편집 이탈:** 뒤로가기로 편집을 나갈 때 "저장하지 않고 나가시겠어요?" 확인 후 홈으로 이동(미저장 시 폐기).
  * **엔딩 마찰 제거:** "완성하고 공유하기"는 결과(절정) 직전에 로그인으로 가로막지 않는다. 게스트는 곧바로 결과 페이지로 이동하고, 로그인 유도는 결과 페이지의 안내 배너로만 처리한다.
* **비회원 처리:** 저장은 게스트 세션 기준으로 임시 저장, 로그인 시 계정에 귀속

### 5.6 최종 결과물 페이지

* **경로:** `/moodboard/:moodboardId`
* **구성:**
  * 상단: 완성된 무드보드 이미지(편집 요소 포함)
  * **무드 성향 이름:** 테스트 여정을 바탕으로 부여되는 명명된 성향 — 성향 이름(한글) + 영문명 + 설명 문구. AI 분석의 **유형명(미학 코어 × 인생 테마)**으로 결정된다(§9). 어느 조합에도 맞지 않으면 기본 성향으로 폴백.
  * **무드 성향 그래프:** 추구미 테스트 여정 기반 **5축 시각화** (고요↔활기 · 따뜻↔서늘 · 미니멀↔맥시멀 · 빈티지↔모던 · 현실↔몽환). 프로토타입에는 ① 레이더 차트, ② 막대 스펙트럼(축별 비중 % + 가장 강한/옅은 결 하이라이트)이 있으나, **그래프 표현 방식(종류·개수)은 아직 확정이 아님**(추후 확정).
  * SNS 공유 버튼 (카카오톡 공유, 링크 복사, 이미지 공유 등 — 지원 채널은 §13 Open Questions로 미확정)
  * 이미지 내보내기 버튼 (PNG 다운로드)
* **동작:**
  * 공유 시 공유용 썸네일/OG 이미지 생성
  * 내보내기 시 캔버스를 이미지 파일로 렌더링하여 다운로드 (프로토타입은 클라이언트 사이드 캔버스 합성으로 타일+스티커+글자를 PNG로 렌더)
  * **재진입 액션:** 편집으로 돌아가기(결과↔편집 왕복), 처음부터 다시 만들기(테스트 재시작), 홈으로.
* **비회원 처리:** 페이지 진입 시 "로그인하면 언제든 다시 볼 수 있어요" 안내 배너 노출 (저장 유도)

### 5.7 세션·상태 보존 (프로토타입 미구현 · 실제 서비스 명세)

> 프로토타입은 인메모리로만 동작하여 새로고침·앱 백그라운드 시 진행 상태가 초기화된다. 이는 플로우 시연용 한계이며, 실제 서비스에서는 아래 정책으로 진행 상태를 보존한다. (프로토타입에서는 코드로 구현하지 않고 명세로만 정의)

* **저장 원칙:** **서버에는 "완성본"만 커밋한다. 진행 중 상태(테스트 답변·편집 중 캔버스)는 모두 클라이언트에 보존하는 드래프트로 취급한다.**
  * ① 로그인/비로그인이 동일한 저장 로직을 공유
  * ② 중간 이탈 시 서버에 무의미한 데이터가 쌓이지 않음
  * ③ 선택마다 서버 왕복이 없어 약전파·오프라인에서도 반응이 즉각적
* **진행 중 테스트:** 각 단계의 선택값(여정 로그)을 클라이언트 store로 관리하고 `localStorage`에 하이드레이션한다. 이탈 후 재진입 시 마지막으로 진행한 단계부터 "이어서 만들기"로 복귀. **테스트 완료 시점에만** 서버에  생성 요청한다.
* **편집 중 보드:** 캔버스 요소(좌표·속성)를 `IndexedDB(or localStorage)`에 주기적으로 autosave하여 새로고침·이탈 후에도 편집 상태를 복원한다. (요소가 많아지고 펜 path가 쌓이면 `localStorage`의 용량·동기 API 한계를 넘으므로 편집 드래프트는 IndexedDB를 사용)
  * **base 이미지 예외:** AI 생성 결과(`base_image_url`)는 서버 에셋이므로 로컬 드래프트에는 두지 않고 **참조(URL/ID)만** 보관한다. 로컬에는 요소 메타데이터 + base 참조만 저장하므로 용량은 작게 유지된다.
  * **서버 저장 시점:** "완성하고 공유하기"(§5.5) 시에만 `Moodboard` row를 서버에 생성한다.  base 이미지 에셋만 서버 스토리지에 존재.
* **회원/게스트 차이:** 드래프트가 클라이언트 기반이므로 **회원·게스트 모두 동일 기기·브라우저에서만 이어하기가 가능**하다. 차이는 커밋된 완성본의 귀속에서만 발생한다.
  * 회원: 완성본(무드보드)을 계정에 영구 귀속. History에서 재열람.
  * 게스트: 완성본은 게스트 세션 기준 임시 저장, 로그인 시 계정에 귀속. 결과 페이지 배너로 로그인(영구 보관)을 유도. 로컬 드래프트는 브라우저 저장소를 비우거나 게스트 세션 만료 시 소실.
* **복원 진입점:** 이어할 로컬 드래프트가 있으면 홈/메인 진입 시 "이어서 만들기" 진입점을 노출한다.

## 11. 비기능 요구사항

* **성능:** 생성 대기 화면은 실제 처리 시간과 무관하게 지루하지 않게 인터랙션·메시지로 체감 대기시간을 줄인다.
* **반응형:** 모바일 우선 (SNS 공유 대상이 모바일일 가능성이 높음).
* **접근성:** 편집 도구(펜/스티커/글자/지우개)는 터치 조작 기준으로 설계.
* **피드백 일관성:** 작업 결과(저장·복사·오류 등)는 공통 토스트로, 확인이 필요한 파괴적 동작(편집 이탈 등)은 확인 모달로 일관되게 처리한다.
* **개인정보:** 카카오 SSO 시 수집 동의 항목(닉네임, 프로필 이미지 등) 명시.
* **콘텐츠 라이선스:** 예시 이미지·스티커 에셋의 저작권/라이선스 확인 필요.
