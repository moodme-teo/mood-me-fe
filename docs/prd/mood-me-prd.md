# PRD: 추구미 무드보드 서비스

## 1. 개요

### 1.1 배경

사용자가 자신이 "추구하는 분위기(추구미)"를 이미지·키워드 선택 기반의 간단한 테스트로 표현하면, AI가 이를 시각적 무드보드로 자동 생성해주고 사용자가 직접 편집·완성하여 SNS에 공유할 수 있는 서비스.

### 1.2 용어 정의

| 용어          | 정의                                                             |
| ------------- | ---------------------------------------------------------------- |
| 추구미        | 사용자가 지향하는 스타일/분위기 (미적 취향)                      |
| 추구미 테스트 | 이미지/키워드 선택을 통해 사용자의 취향을 수집하는 질문 플로우   |
| 무드보드      | 선택 결과를 바탕으로 AI가 생성한, 이미지·색·키워드로 구성된 보드 |
| 비회원        | 카카오 로그인을 하지 않고 서비스를 이용하는 사용자               |

## 2. 목표

- 사용자가 로그인 장벽 없이 즉시 무드보드 생성 경험을 체험할 수 있게 한다.
- AI 생성 대기 시간을 지루하지 않게 만들어 이탈을 최소화한다.
- 결과물(무드보드 + 성향 그래프)의 SNS 공유를 유도해 자연 유입을 만든다.

### 성공 지표 (제안, 추후 확정 필요)

- 추구미 테스트 완료율 (테스트 시작 → 생성 요청)
- 무드보드 저장/내보내기 전환율
- SNS 공유율
- 비회원 → 회원 전환율 (저장/공유 시점 로그인 유도 기준)

## 3. 대상 사용자 및 접근 권한

| 기능                           | 비회원                      | 회원(카카오 SSO) |
| ------------------------------ | --------------------------- | ---------------- |
| 메인 페이지 열람               | ✅                          | ✅               |
| 추구미 테스트 진행             | ✅                          | ✅               |
| 무드보드 생성                  | ✅                          | ✅               |
| 무드보드 편집                  | ✅                          | ✅               |
| 결과 열람 (당일 세션 내)       | ✅                          | ✅               |
| 무드보드 영구 저장/재방문 조회 | ❌ (세션 종료 시 소실 가능) | ✅               |
| SNS 공유 / 이미지 내보내기     | ✅                          | ✅               |

> 비회원도 전체 플로우(테스트→생성→편집→결과·공유·내보내기)를 끝까지 이용 가능하게 하되, 결과물을 나중에 다시 보려면 로그인이 필요하다는 점을 결과 페이지에서 안내한다.

## 4. 전체 유저 플로우

```
[로그인/비회원 진입]
        │
        ▼
   [메인 페이지] ──(무드보드 만들기 클릭)──▶ [추구미 테스트]
                                                  │ (질문 N개 반복)
                                                  ▼
                                          [무드보드 생성중]
                                                  │ (AI 생성 완료)
                                                  ▼
                                            [무드보드 편집]
                                                  │ (저장)
                                                  ▼
                                          [최종 결과물 페이지]
                                          ├─ SNS 공유
                                          └─ 이미지 내보내기
```

## 5. 화면별 상세 명세

### 5.1 로그인

- **경로:** `/login` (또는 메인 진입 시 모달)
- **구성:** 카카오 SSO 로그인 버튼, "비회원으로 계속하기" 버튼
- **동작:**
  - 카카오 로그인 성공 시 → 메인 페이지로 이동, 세션에 사용자 식별자 저장
  - 비회원 선택 시 → 클라이언트에 게스트 세션 ID 발급 후 메인 페이지로 이동
- **예외:** 카카오 인증 실패/취소 시 로그인 화면에 에러 토스트 노출, 재시도 가능

### 5.2 메인 페이지

- **경로:** `/`
- **구성:**
  - 서비스(무드보드) 설명 영역
  - 무드보드 예시 이미지 (캐러셀 또는 그리드, 다른 사용자 결과물 또는 큐레이션 샘플)
  - "무드보드 만들기" CTA 버튼
- **동작:** CTA 클릭 시 추구미 테스트 첫 질문으로 이동, 새 세션(테스트 세션) 생성
- **비회원 처리:** 별도 제약 없음

### 5.3 추구미 테스트

- **경로:** `/test/:sessionId`
- **상단 헤더:**
  - 뒤로가기 버튼 (이전 질문으로 이동, 첫 질문에서는 메인으로)
  - 진척도 표시 (예: `3 / 8`, 프로그레스바)
  - 프리뷰 이미지: 지금까지 선택한 요소들을 조합해 "완성되어 가는" 무드보드 미리보기를 실시간 반영
- **본문:**
  - 질문 텍스트 + 질문 설명(부가 설명/의도)
  - 선택지 UI 3가지 타입 중 질문별로 하나 적용
    1. **이미지만 선택** — 이미지 카드 그리드에서 1개 이상 선택
    2. **텍스트(키워드)만 선택** — 키워드 칩 목록에서 선택
    3. **이미지 선택 + 텍스트로 구체화** — 이미지 선택 후, 해당 이미지를 설명하는 키워드/짧은 텍스트 입력 또는 추가 선택
- **동작:**
  - 선택 완료 시 "다음" 활성화 → 다음 질문으로 이동, 진척도·프리뷰 갱신
  - 마지막 질문 완료 시 → 무드보드 생성중 화면으로 이동, 지금까지의 선택 답변을 생성 요청으로 전송
- **상태 저장:** 뒤로가기로 이전 질문 이동 시 이전 선택값 유지 (재선택 가능)

### 5.4 무드보드 생성중

- **경로:** `/test/:sessionId/generating`
- **구성:**
  - 사용자가 선택한 이미지들이 가상의 무드보드 레이아웃 위에 순차적으로 배치되며 채워지는 인터랙션 애니메이션
  - 프로그레스바
  - 프로그레스바 상단 상태 메시지: "AI가 그리는 중..." 류의 위트있는 문구, 일정 간격(예: 2~3초)마다 로테이션
- **동작:**
  - 백그라운드에서 실제 AI 생성 작업 진행 상태를 폴링/구독하고, 진행률에 맞춰 프로그레스바 갱신
  - 생성 완료 시 자동으로 무드보드 편집 화면으로 이동
- **예외:** 생성 실패 시 에러 화면 + "다시 시도" 버튼 (추구미 테스트 결과는 유지한 채 재생성 요청)

### 5.5 무드보드 편집

- **경로:** `/moodboard/:moodboardId/edit`
- **구성:**
  - AI가 완성한 무드보드 캔버스
  - 편집 도구: 펜(자유 드로잉), 스티커(스티커 팔레트에서 추가), 글자(텍스트 박스 추가/폰트·색상), 드래그(요소 위치 이동/리사이즈)
  - 저장 버튼
- **동작:**
  - 각 도구는 캔버스 위 요소로 추가되며 개별 이동/삭제 가능
  - 저장 시 현재 캔버스 상태(요소 좌표·속성 포함)를 서버에 저장 후 결과물 페이지로 이동
- **비회원 처리:** 저장은 게스트 세션 기준으로 임시 저장, 로그인 시 계정에 귀속

### 5.6 최종 결과물 페이지

- **경로:** `/moodboard/:moodboardId`
- **구성:**
  - 상단: 완성된 무드보드 이미지
  - 하단: 나의 무드 성향 그래프 (추구미 테스트 답변 기반 축 시각화, 예: 레이더 차트)
  - SNS 공유 버튼 (카카오톡 공유, 링크 복사, 이미지 공유 등)
  - 이미지 내보내기 버튼 (PNG/JPG 다운로드)
- **동작:**
  - 공유 시 공유용 썸네일/OG 이미지 생성
  - 내보내기 시 캔버스를 이미지 파일로 렌더링하여 다운로드
- **비회원 처리:** 페이지 진입 시 "로그인하면 언제든 다시 볼 수 있어요" 안내 배너 노출 (저장 유도)

## 9. AI 생성 파이프라인 개요

1. 테스트 세션의 답변(이미지 선택/키워드/커스텀 텍스트)을 취합
2. 답변을 이미지 생성 프롬프트로 조합 (선택 이미지 → 스타일 레퍼런스, 키워드 → 텍스트 프롬프트)
3. 이미지 생성 모델 호출 (비동기 job)
4. 진행률을 클라이언트에 전달 (폴링 또는 SSE/WebSocket)
5. 생성 완료 시 결과 이미지를 `Moodboard.base_image_url`로 저장, 편집 화면으로 전환

## 10. 엣지 케이스 및 에러 처리

| 상황                            | 처리 방안                                                      |
| ------------------------------- | -------------------------------------------------------------- |
| 카카오 로그인 실패/취소         | 에러 토스트, 로그인 화면 유지, 비회원 진입 유도                |
| 테스트 중 이탈 후 재방문        | 회원: 이어하기 가능 / 비회원: 세션 만료 전까지만 유지          |
| 생성 실패                       | 재시도 버튼 제공, 기존 답변 유지                               |
| 생성 시간 과도 지연             | 일정 시간 초과 시 "조금만 더 기다려주세요" 등 안내 메시지 전환 |
| 편집 중 저장 실패               | 로컬 임시 저장(autosave) 후 재시도, 사용자에게 실패 알림       |
| 비회원 세션 만료 후 결과물 접근 | 결과물 소실 안내 + 로그인 시 저장 가능하다는 안내              |
| 이미지 내보내기 실패            | 재시도 옵션 제공                                               |

## 11. 비기능 요구사항

- **성능:** 생성 대기 화면은 실제 처리 시간과 무관하게 지루하지 않게 인터랙션·메시지로 체감 대기시간을 줄인다.
- **반응형:** 모바일 우선 (SNS 공유 대상이 모바일일 가능성이 높음).
- **접근성:** 편집 도구(펜/스티커/글자)는 터치 조작 기준으로 설계.
- **개인정보:** 카카오 SSO 시 수집 동의 항목(닉네임, 프로필 이미지 등) 명시.
- **콘텐츠 라이선스:** 예시 이미지·스티커 에셋의 저작권/라이선스 확인 필요.

## 12. Out of Scope (이번 범위 아님)
