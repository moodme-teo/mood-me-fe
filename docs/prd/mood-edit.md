## PRD: 무드보드 크롭 에디터

> 이 문서는 원래 **범용 이미지 크롭 앱**의 탐색 문서로 쓰였다. 무드미에 들어오면서 범위가
> 크게 줄었다. `mood-me-prd.md` §5.5 와 충돌하면 **이 문서가 최신**이다.

### 1. 제품 개요

**AI가 생성한 무드보드 이미지 1장**을 확대·이동해 구도를 잡고, 도형과 배경을 골라
최종 이미지로 만드는 편집기.

핵심 가치는 **복잡한 디자인 툴 없이도 예쁜 크롭 이미지를 빠르게 만드는 것**이다.

#### 이 편집기가 하지 않는 것

편집 대상은 언제나 **직전 단계에서 AI가 만든 이미지 한 장**이다. 사용자는 사진을
불러오지 않고, 고르지도 않는다. 아래는 원본 탐색 문서에 있었으나 무드미 범위 밖이다.

| 기능                             | 판정     | 이유                                      |
| -------------------------------- | -------- | ----------------------------------------- |
| 이미지 불러오기 / 앨범 선택      | **제외** | 편집 대상은 AI 생성 이미지로 고정         |
| 이미지 썸네일 그리드             | **제외** | 고를 이미지가 하나뿐                      |
| 최근 작업물 / 샘플 템플릿        | **제외** | 홈(History)이 그 역할을 한다              |
| 위젯 사이즈 이미지 · 위젯 템플릿 | P2       | MVP 제외 (§3.6)                           |
| 클립보드 복사                    | P2       | 공유는 링크 복사 + 이미지 저장으로 충분   |
| 이미지 회전                      | P2       | 확대·이동만으로 구도가 잡힌다             |
| 원본 비율 저장                   | P2       | 출력은 정사각형 고정 (CROP_SIZE)          |
| 그라데이션 배경                  | P2       | 미구현                                    |
| 최근 사용 색상                   | P2       | 미구현                                    |
| 텍스트 · 스티커 · 레이어 · 필터  | P2       | #102 에서 캔버스 편집기를 걷어내며 빠졌다 |

이 목록이 흐릿하면 "편집 화면에서 다른 사진을 추가할 수 있는가?"를 팀원마다 다르게
읽는다. **답은 아니오다.**

## 2. 주요 사용자

추구미 테스트를 마치고 자기 무드보드를 받아 든 사람. 별도의 사용자 유형은 없다 —
이 화면은 핵심 여정의 한 칸이지 독립 제품이 아니다.

## 3. 핵심 기능

- 이미지 확대/축소
- 이미지 이동

---

### 3.2 크롭 도형 선택

사용자는 이미지를 원하는 도형으로 자를 수 있다.

기본 제공 도형은 다음과 같다.

- **크롭 안 함** (기본값)
- 원
- 타원
- 정사각형
- 둥근 사각형
- 세로 캡슐
- 가로 캡슐
- 별
- 하트
- 다이아몬드

도형 선택 UI는 하단 가로 스크롤 형태로 제공한다.
선택한 도형은 상단 미리보기 영역에 즉시 반영된다.

**기본값은 "크롭 안 함" 이다.** AI가 만든 무드보드는 세로형 전체 콜라주라, 진입하자마자
원형으로 잘려 있으면 사용자는 **생성 결과가 망가졌다고 오해한다.** 자르는 것은 사용자가
고르는 행동이어야지, 화면이 먼저 저지르는 일이 아니다.

> 현재 구현은 `circle` + 투명 배경이다 (`useCropEditor.ts`의 `DEFAULT_STATE`).
> 변경은 [#117](https://github.com/moodme-teo/mood-me-fe/issues/117).

---

### 3.3 투명 배경

사용자는 배경을 투명하게 설정할 수 있다.

투명 배경 선택 시 체크보드 패턴으로 표시한다.
저장 시 PNG 형식으로 저장되어야 한다.

필수 조건:

- 투명 배경 미리보기 제공
- PNG 저장 지원
- 투명 영역 유지
- 배경색과 투명 배경 간 전환 가능

> 체크보드는 Konva Stage가 아니라 그 뒤에 깔린 DOM `div`다. **export에는 들어가지 않는다.**

---

### 3.4 배경색 설정

사용자는 크롭된 이미지 뒤의 배경색을 선택할 수 있다.

배경 옵션은 다음과 같다.

- 투명
- 이미지 블러
- 흰색
- 검정 (`#171717`)
- 사용자 지정 컬러
- 이미지 기반 자동 추출 색상 (§3.5)

그라데이션과 최근 사용 색상은 P2다.

---

### 3.5 Dynamic Colors

생성된 이미지에서 주요 색상을 자동 추출해 배경색 후보로 제공한다.

기능 요구사항:

- 이미지 dominant color 추출 (canvas pixel sampling)
- 5~8개 컬러 팔레트 생성
- 선택 즉시 배경에 반영

**추출에 실패해도 에러 화면을 띄우지 않는다.** 팔레트는 편집을 막는 기능이 아니다.
색을 못 뽑으면 그 영역만 조용히 접고 안내 한 줄을 남긴다.

> 추천 색상을 불러오지 못했어요. 기본 색상은 그대로 사용할 수 있어요.

---

### 3.6 위젯용 이미지 제작

**P2 — MVP 제외.** 여러 크롭 이미지 배치, 위젯 사이즈 프리셋, 텍스트 추가,
시간/날짜 템플릿. 무드미 MVP는 단일 이미지 크롭만 지원한다.

## 4. 화면 구성

### 4.1 진입

홈 화면은 없다. 진입 경로는 둘뿐이다.

- `/test/[sessionId]/edit` — 생성 직후, 결과를 처음 다듬는다
- `/moodboard/[moodboardId]/edit` — 저장된 보드를 다시 다듬는다

두 경로 모두 같은 `MoodboardCropEditor`를 렌더한다.

---

### 4.2 편집 화면

앱의 핵심 화면.

상단:

- 뒤로
- "편집"
- **완료**

중앙:

- 편집 미리보기 영역
- 현재 도형으로 크롭된 이미지
- 배경색 또는 투명 배경

하단 모드 탭:

- 이미지 (확대 슬라이더 · 구도 초기화)
- 도형 (가로 스크롤 도형 리스트)
- 배경 (투명 · 블러 · 흰색 · 검정 · 직접 선택)
- 색상 (추천 팔레트)

---

### 4.3 완료

편집 화면의 **완료**는 오직 하나만 한다 — 편집 결과를 커밋하고 결과 페이지로 이동한다.
이미지를 내려받는 일은 결과 페이지의 몫이다 (§4.4).

> **용어를 나눈 이유.** "저장"이 세 가지를 동시에 가리키고 있었다. 편집을 끝낸다,
> 서버에 커밋한다, 파일을 내려받는다. 사용자는 "저장"을 누르면 앨범에 뭔가 생겼다고
> 기대한다. 그래서 편집 화면에서는 **완료**만 남기고, 파일로 만드는 행동은
> 결과 페이지의 **이미지 저장**으로 모은다.

> 현재 구현은 헤더가 "저장"이고, 그 시트 안에 PNG/JPG 다운로드와 "완성하고 공유하기"가
> 함께 있다. 변경은 [#118](https://github.com/moodme-teo/mood-me-fe/issues/118).

---

### 4.4 결과 페이지의 이미지 저장

결과 페이지(`mood-me-prd.md` §5.6)의 액션은 **이미지 저장**과 **SNS 공유**다.

**이미지 저장**을 누르면 형식을 고른다.

- PNG로 저장 — 투명값 유지
- JPG로 저장 — 투명 영역을 흰색으로 변환

## 7. 기능 상세 요구사항

### 이미지 편집

- 사용자는 이미지를 핀치 줌으로 확대/축소할 수 있다 (`MIN_ZOOM` 1 ~ `MAX_ZOOM` 6).
- 드래그로 이미지 위치를 조정할 수 있다.
- 더블탭 시 기본 위치로 초기화된다.
- 이미지를 프레임 밖으로 밀어낼 수 없다 (`clampTransform`).

**빈 공간 규칙은 도형에 따라 다르다** (`getCropFit`).

- 도형을 골랐으면 `cover` — zoom=1에서 이미지가 프레임을 꽉 채운다. 빈 공간이 생길 수 없다.
- **크롭 안 함**이면 `contain` — 이미지 전체가 보이고, 남는 여백은 배경색으로 채워진다.
  기본값이 "크롭 안 함"인 이유가 이것이다. 생성 결과를 자르지 않고 통째로 보여준다.

### 도형 크롭

- 도형 변경 시 이미지 위치와 확대 상태는 유지된다.
- 도형 선택 즉시 미리보기에 반영된다.
- 도형을 빠르게 연달아 바꿔도 마지막 선택만 적용된다.
- 도형 영역 밖 이미지는 마스킹 처리한다.

### 배경

- 배경은 크롭 이미지 뒤에 적용된다.
- 투명 배경 선택 시 체크보드 패턴을 보여준다.
- JPG 저장 시 투명 배경은 흰색으로 변환한다.
- PNG 저장 시 투명값을 유지한다.

### 이탈

- 편집 중 이탈해도 **드래프트는 보존한다** (IndexedDB `mood-me` / `moodboard-drafts`).
- 드래프트를 지우는 것은 사용자가 명시적으로 "처음부터 다시 만들기"를 고를 때뿐이다.
- 이탈 경로는 화면 뒤로 버튼 · 브라우저 뒤로가기 · 홈 버튼 · 새로고침 · 탭 닫기.

> 현재 구현은 뒤로 버튼만 확인 다이얼로그를 띄우고 "폐기"라고 안내한다. 그런데 실제로는
> IndexedDB에 autosave되어 홈의 "이어서 만들기"로 복원된다. **안내와 동작이 어긋난다.**
> 정책 정리는 [#119](https://github.com/moodme-teo/mood-me-fe/issues/119).

모달 문구:

> 편집을 멈출까요? 지금까지 수정한 내용은 이 기기에 임시 저장돼요.
> [계속 편집] [나가기]

## 11. 미리보기와 저장 결과물

프론트 구현의 핵심은 **마스킹 + 캔버스 export**다.

- 편집 미리보기: Konva `Group`의 `clipFunc`
- 실제 저장: `stage.toDataURL({ pixelRatio })`
- 도형 마스크: SVG path
- 색상 추출: canvas pixel sampling
- 투명 PNG: `mimeType: "image/png"`
- JPG: `compositeOnWhite()` → `toDataURL("image/jpeg", 0.92)`

**"미리보기에서 보이는 것"과 "저장 결과물"이 달라지면 신뢰가 깨진다.**

이 계약은 문장으로만 두지 않는다. E2E가 픽셀로 지킨다 (`docs/convention/qa.md` §픽셀 단언).

최소 검증 대상:

- 선택한 도형이 export에 적용된다
- 이미지 위치·배율이 같다
- 선택한 배경색이 같다
- 투명 PNG의 알파값이 유지된다
- JPG는 투명 영역을 흰색으로 변환한다
- 출력 크기는 `CROP_SIZE × EXPORT_PIXEL_RATIO` (720×720)로 고정된다 — 화면 DPR과 무관하다

## 12. 재편집을 위한 편집 상태

결과 페이지에서 "편집"으로 돌아가면 **직전 구도가 복원돼야 한다.**

지금 저장되는 것은 평면 export 이미지(`exported_image_data_url`)뿐이라, 다시 편집하면
항상 기본값으로 초기화된다. 복원하려면 편집 상태를 함께 저장해야 한다.

```ts
edit_state: {
  sourceImageUrl: string;
  shapeId: string;
  background: BackgroundOption;
  scale: number;
  x: number;
  y: number;
}
```

데이터 모델 추가가 필요하다. [#116](https://github.com/moodme-teo/mood-me-fe/issues/116).

## 13. 에러 · 엣지 케이스

에러 화면을 띄울 것과, 조용히 넘어갈 것을 구분한다.

| 상황                                | 처리                                              |
| ----------------------------------- | ------------------------------------------------- |
| 원본 이미지 로드 실패               | 에러 + 다시 시도                                  |
| CORS로 canvas tainted → export 실패 | 에러 토스트. 같은 출처 이미지만 쓰므로 발생 안 함 |
| 추천 색상 추출 실패                 | **팔레트 영역만 접는다.** 편집은 계속             |
| 추천 팔레트가 전부 유사색           | 그대로 노출 (판단하지 않는다)                     |
| 매우 밝거나 어두운 이미지           | 그대로 노출                                       |
| 세로로 매우 긴 / 저해상도 이미지    | `clampTransform`이 최소 배율을 보장               |
| EXIF 회전 정보                      | 브라우저 디코더에 위임                            |
| 직접 선택한 색상의 잘못된 포맷      | `<input type="color">`가 막는다                   |
| 투명 배경에서 JPG 선택              | 흰색으로 변환 (안내 문구 노출)                    |
| 저장 도중 페이지 이탈               | 드래프트 보존 (§7 이탈)                           |
| 완성본 저장 실패                    | 편집 상태 유지 + 재시도. `upsert`라 중복 row 없음 |

문구:

> 완성본을 저장하지 못했어요. 편집 내용은 그대로 남아 있어요.
