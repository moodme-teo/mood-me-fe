**이미지를 원하는 형태로 잘라서 배경/색상/위젯용 이미지로 만드는 비주얼 크롭 에디터**에 가까워 보여.

## PRD: 이미지 크롭 앱

### 1. 제품 개요

사용자가 사진을 불러와 원형, 타원형, 별, 둥근 사각형 등 다양한 형태로 크롭하고, 배경색이나 투명 배경을 적용해 저장할 수 있는 모바일 이미지 편집 앱.

핵심 가치는 **복잡한 디자인 툴 없이도 예쁜 크롭 이미지를 빠르게 만드는 것**이다.
SNS 이미지, 위젯 배경, 프로필 이미지, 스티커, 썸네일 제작에 활용할 수 있다.

---

## 2. 주요 사용자

1. SNS용 이미지나 프로필 이미지를 자주 만드는 사용자
2. 아이폰 위젯, 배경화면, 노션 커버 등 꾸미기 이미지를 만드는 사용자
3. 디자인 툴은 부담스럽지만 감각적인 이미지를 만들고 싶은 사용자
4. 투명 배경 PNG, 커스텀 도형 이미지가 필요한 사용자

---

## 3. 핵심 기능

- 이미지 확대/축소
- 이미지 이동
- 이미지 회전

---

### 3.2 크롭 도형 선택

사용자는 이미지를 원하는 도형으로 자를 수 있다.

기본 제공 도형은 다음과 같다.

- 원
- 타원
- 정사각형
- 둥근 사각형
- 세로 캡슐
- 가로 캡슐
- 별
- 하트
- 다이아몬드
- 커스텀 불규칙 도형

도형 선택 UI는 하단 가로 스크롤 형태로 제공한다.
선택한 도형은 상단 미리보기 영역에 즉시 반영된다.

---

### 3.3 투명 배경

사용자는 배경을 투명하게 설정할 수 있다.

투명 배경 선택 시 체크보드 패턴으로 표시한다.
저장 시 PNG 형식으로 저장되어야 한다.

필수 조건:

- 투명 배경 미리보기 제공
- PNG 저장 지원
- 투명 영역 유지
- 배경색과 투명 배경 간 전환 가능

---

### 3.4 배경색 설정

사용자는 크롭된 이미지 뒤의 배경색을 선택할 수 있다.

배경 옵션은 다음과 같다.

- 투명
- 흰색
- 검정색
- 이미지 기반 자동 추출 색상
- 그라데이션
- 사용자 지정 컬러
- 최근 사용 색상

레퍼런스처럼 이미지에서 추출한 톤을 팔레트로 보여주면 좋다.
예를 들어 토마토 사진을 선택하면 빨강, 초록, 베이지 계열이 추천되는 방식.

---

### 3.5 Dynamic Colors

선택한 이미지에서 주요 색상을 자동 추출해 배경색 후보로 제공한다.

사용자는 이미지 분위기에 맞는 배경색을 빠르게 고를 수 있다.

기능 요구사항:

- 이미지 dominant color 추출
- 5~8개 컬러 팔레트 생성
- 밝은색/어두운색 구분
- 선택 즉시 배경에 반영
- 이전 선택 색상 저장

---

### 3.6 위젯용 이미지 제작

레퍼런스의 “Widgets” 화면처럼 사용자가 크롭 이미지를 여러 개 배치해 위젯 스타일 이미지를 만들 수 있다.

MVP에서는 단일 이미지 크롭만 지원하고, 이후 확장 기능으로 제공해도 된다.

확장 기능:

- 여러 크롭 이미지 배치
- 위젯 사이즈 프리셋
- 배경 이미지 설정
- 텍스트 추가
- 시간/날짜 템플릿
- 저장용 캔버스 출력

---

## 4. 화면 구성

### 4.1 홈 화면

사용자가 바로 이미지를 선택할 수 있는 진입 화면.

구성:

- 앱 타이틀
- “이미지 선택하기” 버튼
- 최근 작업물
- 샘플 템플릿

---

### 4.2 편집 화면

앱의 핵심 화면.

상단:

- 상태바
- 앱 타이틀
- 저장/다운로드 버튼

중앙:

- 편집 미리보기 영역
- 현재 도형으로 크롭된 이미지
- 배경색 또는 투명 배경

하단:

- 모드 탭

  - 이미지
  - 도형
  - 배경
  - 색상

- 이미지 썸네일 그리드
- 도형 선택 리스트
- 컬러 팔레트

---

### 4.3 저장 화면

사용자가 결과물을 저장하거나 공유할 수 있다.

저장 옵션:

- PNG
- JPG
- 투명 배경 PNG
- 정사각형 이미지
- 원본 비율 이미지
- 위젯 사이즈 이미지

공유 옵션:

- 앨범 저장
- 클립보드 복사
- SNS 공유

---

## 5. MVP 범위

첫 버전에서 꼭 필요한 기능은 이 정도면 충분하다.

1. 이미지 불러오기
2. 이미지 확대/축소/이동
3. 기본 도형 크롭
4. 배경색 선택
5. 투명 배경 PNG 저장
6. 이미지 색상 기반 추천 팔레트
7. 결과물 저장

MVP에서 제외해도 되는 기능:

- 여러 이미지 동시 배치
- 텍스트 추가
- 위젯 템플릿
- 커스텀 도형 직접 그리기
- 레이어 기능
- 고급 필터

처음부터 포토샵 작은 괴물을 만들면 앱이 뚱뚱해질 수 있어서, 1차는 “예쁜 도형 크롭 + 투명 저장”에 집중하는 게 좋아 보여.

---

## 6. 사용자 플로우

사용자는 앱을 열고 이미지를 선택한다.
편집 화면에서 도형을 고른다.
이미지를 손가락으로 이동하거나 확대해서 원하는 구도를 맞춘다.
배경을 투명, 단색, 추천 색상 중에서 선택한다.
저장 버튼을 누르고 PNG 또는 JPG로 저장한다.

핵심 플로우는 5단계 이내로 끝나야 한다.

---

## 7. 기능 상세 요구사항

### 이미지 편집

- 사용자는 선택한 이미지를 핀치 줌으로 확대/축소할 수 있다.
- 드래그로 이미지 위치를 조정할 수 있다.
- 더블탭 시 기본 위치로 초기화된다.
- 이미지가 크롭 영역 밖으로 비어 보이지 않도록 최소 확대값을 제한한다.

### 도형 크롭

- 도형 변경 시 이미지 위치와 확대 상태는 유지된다.
- 도형 선택 즉시 미리보기에 반영된다.
- 도형 영역 밖 이미지는 마스킹 처리한다.

### 배경

- 배경은 크롭 이미지 뒤에 적용된다.
- 투명 배경 선택 시 체크보드 패턴을 보여준다.
- JPG 저장 시 투명 배경은 흰색으로 변환한다.
- PNG 저장 시 투명값을 유지한다.

### 저장

- 저장 전 최종 미리보기를 제공한다.
- 저장 성공 시 토스트 메시지를 보여준다.
- 저장 실패 시 권한 안내를 제공한다.

---

## 8. 비기능 요구사항

- 편집 반응은 즉시 느껴져야 한다.
- 이미지 확대/이동 시 끊김이 없어야 한다.
- 저장 결과물은 미리보기와 최대한 동일해야 한다.
- 앱 최초 사용자는 설명 없이도 1분 안에 결과물을 만들 수 있어야 한다.
- iOS 기준 앨범 접근 권한 처리가 필요하다.
- Android 지원 시 저장소 권한 정책을 별도로 고려한다.

---

## 9. 성공 지표

- 첫 이미지 저장 완료율
- 이미지 선택 후 저장까지 걸린 평균 시간
- 도형 변경 사용률
- 투명 배경 저장 비율
- 추천 색상 선택률
- 재방문율
- 저장 후 공유 버튼 클릭률

## 11. 개발 관점 메모

프론트 구현 기준으로는 핵심이 **마스킹 + 캔버스 export**야.

웹/앱 하이브리드라면 다음 구조가 적합해 보인다.

- 편집 미리보기: CSS mask 또는 SVG clipPath
- 실제 저장: Canvas 렌더링
- 도형 마스크: SVG path 관리
- 색상 추출: canvas pixel sampling
- 투명 PNG: canvas.toBlob image/png
- 제스처: pinch/drag gesture 라이브러리 활용

주의할 점은 “미리보기에서 보이는 것”과 “저장 결과물”이 달라지면 신뢰가 깨진다는 것.
그래서 최종 export 로직을 초반부터 잡는 게 중요하다. 이 앱의 엔진룸은 예쁘게 생긴 도형 버튼이 아니라, 조용히 돌아가는 캔버스 한 장이다.
