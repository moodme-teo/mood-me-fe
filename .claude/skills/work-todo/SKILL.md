# /work-todo — todo item 추가 (번호 자동 부여)

`docs/work/plan/todo-list.md` 에 새 todo item을 추가한다. **번호는 전역 고유값으로 이 스킬이 자동 부여**한다 — 사용자가 번호를 지정해도 무시하고 카운터 기준으로 부여한다.

사용 예: `/work-todo 빌더, 로그인 페이지 구현, 26.07.06~26.07.12, 로그인`

> 로컬 파일 수정만 하는 스킬. gh 불필요. 커밋은 사용자 몫.

## 1. 입력 파싱

입력에서 다음 4개 값을 파싱한다 (구분자는 쉼표/줄바꿈 등 유연하게 해석):

- **역할**: `프로토타이퍼 | 빌더 | 스위퍼 | 그로워 | 메인테이너 | UXUI Leader | PM`
- **작업이름**: 자유 텍스트
- **일정**: `26.07.06~26.07.12` 형식 (`YY.MM.DD~YY.MM.DD`). 하루짜리면 `26.07.06~26.07.06`.
- **페이지**: `로그인 | 메인 | 보드생성 | 보드완성 | 보드편집 | 테스트 | 공통`

빠졌거나 허용값이 아닌 항목은 `AskUserQuestion` 으로 확인해 확정한다(역할·페이지는 선택지 제시, 일정은 형식 검증). 여러 item을 한 번에 입력하면 각각 파싱해 순서대로 처리한다.

## 2. 번호 부여

`docs/work/plan/todo-list.md` 상단의 카운터 줄을 읽는다:

```
> **마지막 부여 번호: a7** — …
```

- 다음 번호 = 마지막 부여 번호 + 1 (예: `a7` → `a8`). 여러 개면 연속 부여 (`a8`, `a9`, …).
- **번호는 절대 재사용하지 않는다.** 테이블에서 항목이 삭제됐더라도 카운터는 되돌리지 않는다.
- 카운터 줄이 없거나 손상됐으면, 문서 전체 테이블에서 가장 큰 `a<N>` 을 찾아 그 다음 번호를 쓰고 카운터 줄을 복구한다.

## 3. 테이블에 행 추가

해당 **역할 섹션**의 테이블 마지막에 행을 추가한다. 이슈 열은 비워둔다:

```markdown
| a8 | 로그인 페이지 구현 | 26.07.06~26.07.12 | 로그인 | |
```

카운터 줄의 번호를 마지막으로 부여한 번호로 갱신한다.

## 4. 결과 보고

- 다음 단계: `/work-issue a8` 로 이슈 문서 초안 작성 + GitHub 이슈 등록
